# GEO/SEO 搜索可见度控制台 — 使用手册

> 本手册面向「不想学习、没耐心、脾气不好」的用户。每一个步骤都有截图说明，看不懂就骂产品经理。

**系统要求**：Chrome / Edge / Safari 最新版
**推荐分辨率**：1920×1080 及以上，**不支持手机**

**常见登录失败**：

| 错误信息 | 原因 | 解决办法 |
|---|---|---|

| 页面一片空白 | server 没启动 | 确保 `pnpm dev` 在跑 |

## 三、第一步：先选项目

**几乎所有功能都要先选「项目」。**

## 四、页面诊断（给网站打分）

### 4.1 什么是页面诊断

输入任意网址，系统会自动抓取页面，分析 SEO 问题，给出一个 0-100 的分数。

### 4.2 操作步骤

1. 左侧菜单点击「**03 页面诊断**」
2. 点击右上角绿色按钮「**+ new_audit**」

```
┌─────────────────────────────────────────────┐
│  // M02 — New Diagnostic                    │
│  Audit Page                                │
│                                             │
│  project_id *                               │
│  › seed-project-1           ← 一般已自动填好│
│                                             │
│  url *                                       │
│  › https://example.com        ← 填网址      │
│                                             │
│  [→ execute]                 ← 点这个        │
└─────────────────────────────────────────────┘
```

3. 填入网址，例如 `https://example.com`
4. 点击「**execute →**」
5. 等待 10-30 秒（取决于网站速度）

### 4.3 看懂诊断结果

分数颜色含义：

| 分数 | 颜色 | 含义 |
|---|---|---|
| 80-100 | 🟢 绿色 | 优秀，没什么大问题 |
| 60-79 | 🟡 黄色 | 一般，有改进空间 |
| 0-59 | 🔴 红色 | 很差，需要立刻优化 |

点击分数可以查看详细问题列表，每个问题都有「高/中/低」等级和修复建议。

## 五、关键词管理

### 5.1 什么是关键词

你想让用户搜什么词时找到你，这些词就是关键词。例如：「SEO 工具」「企业关键词优化」。

### 5.2 添加关键词

1. 左侧菜单点击「**04 关键词**」
2. 确认右上角已选对项目
3. 点击「**+ new_kw**」

```
┌──────────────────────────────────────┐
│  // M06 — New Keyword                │
│  Add Keyword                         │
│                                      │
│  keyword *                           │
│  › SEO optimization tool  ← 填关键词  │
│                                      │
│  intent     priority (1-5)            │
│  [ INFORMATIONAL ▼]  [ 3 ]           │
│  ← 意图         ← 优先级 1最高 5最低  │
│                                      │
│  target_url (optional)               │
│  › https://...                       │
│                                      │
│  [cancel]          [create →]        │
└──────────────────────────────────────┘
```

**意图（intent）**说明：
- `INFORMATIONAL` — 用户想了解信息，如「什么是 GEO」
- `COMMERCIAL` — 用户在对比选择，如「最好的 SEO 工具」
- `TRANSACTIONAL` — 用户想完成交易，如「买 SEO 软件」
- `NAVIGATIONAL` — 用户想找特定网站，如「Ahrefs 官网」
- `LOCAL` — 本地搜索，如「上海 SEO 公司」
- `COMPARISON` — 用户在对比多个选项

**优先级**：
- P1 = 最高优先，最重要
- P3 = 默认
- P5 = 最低

### 5.3 批量导入

点击「**import_csv**」按钮，粘贴 CSV 格式数据：

```csv
text,intent,priority
SEO optimization,INFORMATIONAL,1
GEO monitoring,COMMERCIAL,2
keyword research,INFORMATIONAL,3
```

## 六、GEO 监测（最重要）

### 6.1 什么是 GEO

GEO = Generative Engine Optimization，让 AI（如 ChatGPT、Perplexity）能够正确引用你的品牌和内容。

### 6.2 管理 GEO 问题

1. 左侧菜单点击「**05 GEO 监测**」
2. 问题 = 用户可能在 AI 搜索引擎里问的问题

添加问题示例：
- 「最好的 SEO 工具是哪些？」
- 「如何提高网站在 AI 搜索中的可见度？」

### 6.3 触发 GEO 监测（跑一次检测）

1. 在 GEO 监测页面，点右上角「**触发监测**」按钮
2. 系统会向 Perplexity、Kimi、抖音等多个渠道发起查询
3. 等待 2-5 分钟（取决于渠道响应速度）
4. 在「**06 GEO 运行**」页面看结果

### 6.4 看懂 GEO 评分

「控制台」页面左上角的 **A · GEO评分** 就是你的分数。

- **分数越高** = AI 越容易引用你的品牌
- **brand_hit** = 你的品牌在 AI 回答中被提到的次数
- **questions** = 你追踪的问题总数

## 七、内容优化（AI 帮你写内容）

### 7.1 分析现有页面

1. 左侧菜单点击「**07 内容优化**」
2. 选「**url_crawl**」模式
3. 填入网址和关键词
4. 点「**→ analyze**」

系统会告诉你：
- 标题和描述是否需要改
- 缺少哪些关键词
- 建议加哪些 FAQ
- 建议加哪些定义段落

### 7.2 生成新内容

1. 左侧菜单进入「**08 内容草稿**」
2. 点「**+ new_draft**」
3. 填主题和目标关键词
4. AI 自动生成 1500+ 字长文

## 八、任务系统

### 8.1 什么是任务

系统会根据页面诊断、GEO 监测、内容分析自动生成优化任务，你也可以手动创建。

### 8.2 查看任务

左侧菜单点击「**11 优化任务**」

页面顶部有 5 个状态标签，点击切换：

```
[ 全部 (5) ] [ todo (2) ] [ 进行中 (1) ] [ 待审 (1) ] [ 完成 (0) ] [ 忽略 (1) ]
```

### 8.3 改变任务状态

找到任务 → 点击状态栏 → 选择新状态：

```
[todo ▾] → 选「doing」= 开始做
[todo ▾] → 选「done」 = 完成
```

### 8.4 手动创建任务

1. 点右上角「**+ new_task**」
2. 填项目 ID、标题、描述（可选）、URL（可选）、优先级
3. 点「**create →**」

## 九、报告

### 9.1 生成报告

1. 左侧菜单点击「**12 报告**」
2. 确认右上角选对项目
3. 点「**+ generate**」
4. 选择类型：
   - **Weekly Report** = 过去 7 天汇总
   - **Monthly Report** = 过去 30 天汇总 + 趋势

### 9.2 查看和下载报告

报告生成后，在列表里：
- 点「**view**」= 在新标签页打开阅读
- 点「**download**」= 下载 Markdown 文件

## 十二、常见问题

### Q: 点了按钮没反应

**原因**：99% 是没选项目。页面右上角那个下拉框，先选项目。

### Q: GEO 评分是 0

**原因**：还没跑过 GEO 监测。先去 GEO 监测页面触发一次。

### Q: 页面显示「参数错误」

**原因**：输入格式不对。检查：
- URL 要带 `https://`
- 关键词至少填一个
- 草稿内容至少 50 个字

### Q: 登录不了，一直显示 loading

**原因**：后端服务没启动。

打开终端，输入：
```bash
docker compose up -d postgres redis   # 先确保数据库在跑
pnpm dev                              # 再启动网站
```

### Q: 想回到首页

**原因**：左侧菜单最顶上的「**01 控制台**」
